= RAZE-TUI Roadmap
:author: Jonathan D.A. Jewell
:toc:
:toc-placement!:
:icons: font

Development roadmap for RAZE-TUI, a polyglot TUI framework with dual formal
verification (Idris2 ABI proofs + SPARK implementation proofs).

toc::[]

== Design Principles

These principles guide every phase:

. **Proof before code** -- Idris2 ABI specs and SPARK contracts are written
  _before_ implementation. The types define the program.
. **Zig is a pure bridge** -- The Zig layer contains zero business logic. It
  translates types and manages lifetimes. Nothing else.
. **SPARK is the core** -- All state management, layout computation, and event
  dispatch live in SPARK-proven Ada. This is where correctness matters most.
. **Rust is a consumer, not the core** -- Rust provides ecosystem integration
  and async support. It does not own state.
. **No runtime errors by construction** -- Between Idris2 (interface) and SPARK
  (implementation), the goal is zero classes of runtime error in the TUI core.

== Non-Goals

* **Not a Ratatui clone** -- RAZE-TUI has a fundamentally different architecture
  (SPARK core vs Rust core). API similarity is incidental, not a goal.
* **Not a general-purpose widget toolkit** -- Focus is on TUI. No GUI backend.
* **Not cross-compiled to WASM** -- Terminal I/O does not translate to browser.
  A future web terminal emulator could consume the widget tree, but that is
  post-1.0.
* **Not dynamically extensible** -- Widgets are compiled in. Plugin systems add
  attack surface incompatible with formal verification.

== Phase 1: Idris2 ABI + SPARK Core + Zig FFI Restructure [CURRENT]

*Milestone: v0.1.0 -- "Verified Foundation"*

=== 1.1 Idris2 ABI Specification

* [ ] Create `src/abi/Types.idr` -- dependent type definitions for all shared types
* [ ] Create `src/abi/Layout.idr` -- memory layout proofs (struct sizes, field offsets)
* [ ] Create `src/abi/Foreign.idr` -- FFI function signature declarations
* [ ] Prove backward compatibility: adding a field cannot break existing callers
* [ ] Platform-specific ABI selection (LP64, LLP64, ILP32) via compile-time evidence
* [ ] C header generation from Idris2 specs to `generated/abi/raze_abi.h`

=== 1.2 SPARK Core Migration

* [ ] Move state management from Zig into SPARK packages
* [ ] SPARK state machine for TUI lifecycle (`Uninitialized -> Running -> Shutdown`)
* [ ] Prove absence of runtime errors in state transitions
* [ ] SPARK flow analysis on all public package interfaces
* [ ] Data flow contracts: every output depends only on declared inputs

=== 1.3 Zig Bridge Refactor

* [ ] Strip business logic from Zig bridge (move to SPARK)
* [ ] Zig imports generated C headers from Idris2
* [ ] Bridge becomes pure type marshalling + lifetime management
* [ ] Rust consumer calls through Zig C ABI to SPARK core

=== 1.4 Contract Correspondence

* [ ] Map Idris2 dependent types to SPARK Pre/Post conditions
* [ ] Document proof coverage in `docs/architecture/ARCHITECTURE.adoc`
* [ ] Health zone reporting: which properties are proven where

== Phase 2: Terminal Backend [PLANNED]

*Milestone: v0.2.0 -- "Terminal Native"*

=== 2.1 Terminal Detection and Initialization (SPARK)

* [ ] Detect terminal capabilities (color depth, Unicode support)
* [ ] Raw mode entry/exit as SPARK state machine (proven transitions)
* [ ] Alternate screen buffer management
* [ ] Signal handling: SIGWINCH (resize), SIGINT/SIGTERM (graceful shutdown)

=== 2.2 ANSI Rendering Engine (SPARK)

* [ ] ANSI escape sequence generation with proven correctness
* [ ] Double buffering: front buffer (displayed) vs back buffer (building)
* [ ] Dirty region tracking -- only redraw changed cells
* [ ] Cell model: character + foreground + background + attributes
* [ ] Unicode grapheme cluster support (width calculation)

=== 2.3 Input Parsing (SPARK)

* [ ] Keyboard input: standard keys, function keys, modifier combinations
* [ ] Mouse input: clicks, scroll, drag, movement (SGR mouse protocol)
* [ ] Bracketed paste mode detection
* [ ] UTF-8 multi-byte sequence parsing (proven no partial reads)
* [ ] Input event queue with bounded capacity (proven no overflow)

=== 2.4 Platform Abstraction (Zig)

* [ ] POSIX terminal ioctl via Zig (thin wrapper, no logic)
* [ ] Windows Console API support via Zig (future)
* [ ] terminfo database reading (capability strings)

== Phase 3: Widget System [PLANNED]

*Milestone: v0.3.0 -- "Widget Tree"*

=== 3.1 Layout Engine (SPARK Verified)

* [ ] Constraint-based layout solver with proven termination
* [ ] Flexbox-style main axis / cross axis distribution
* [ ] Absolute and relative positioning
* [ ] Min/max size constraints (proven satisfiable)
* [ ] Margin, padding, border box model
* [ ] Layout cache with version-based invalidation

=== 3.2 Core Widgets (SPARK Verified)

* [ ] **Box** -- bordered container with title, SPARK-proven bounds
* [ ] **Text** -- static text with word wrap, alignment, truncation
* [ ] **Input** -- single-line text input, cursor, selection range
* [ ] **List** -- scrollable item list with selection tracking
* [ ] **Table** -- column definitions, sorting, row selection
* [ ] **Tabs** -- tabbed container with keyboard navigation

=== 3.3 Widget Tree Model

* [ ] Immutable widget tree (rebuild on state change, like Elm/React)
* [ ] Diff algorithm: compare previous and current tree, emit minimal updates
* [ ] Focus chain: automatic tab-order from tree structure
* [ ] Widget identity for animation and state preservation

== Phase 4: Rust Async Integration [PLANNED]

*Milestone: v0.4.0 -- "Async Bridge"*

=== 4.1 Event Loop Bridge

* [ ] Tokio integration: bridge SPARK event dispatch to Rust async tasks
* [ ] Smol integration: lightweight alternative for smaller binaries
* [ ] Channel-based communication: Rust async -> SPARK event queue
* [ ] Timer events via Rust async (debounce, animation frames)

=== 4.2 Rust API Surface

* [ ] Builder pattern for widget construction from Rust
* [ ] Trait-based rendering: `impl Widget for MyWidget`
* [ ] State management: Rust owns application state, SPARK owns TUI state
* [ ] Error handling: SPARK proof failures surface as Rust `Result` types

=== 4.3 Crate Publishing

* [ ] `raze-core` crate on crates.io (types only)
* [ ] `raze-tui` crate on crates.io (full TUI with bundled SPARK/Zig)
* [ ] Alire package for Ada consumers
* [ ] Zig package for Zig package manager

== Phase 6: Accessibility [PLANNED]

*Milestone: v0.6.0 -- "Accessible by Default"*

=== 6.1 Screen Reader Support

* [ ] ARIA-like role annotations on all widgets
* [ ] Cursor routing for screen reader focus tracking
* [ ] Announcements: live region protocol for status changes
* [ ] Structured navigation: headings, landmarks, lists

=== 6.2 Visual Accessibility

* [ ] High contrast theme (WCAG AAA contrast ratios)
* [ ] Large text mode (2x cell rendering)
* [ ] Reduced motion mode (disable animations)
* [ ] Color-blind safe palette options (protanopia, deuteranopia, tritanopia)

=== 6.3 Keyboard-Only Navigation

* [ ] Full keyboard operability: every action reachable without mouse
* [ ] Visible focus indicator on all focusable widgets
* [ ] Keyboard shortcuts discoverable via help overlay
* [ ] Spatial navigation (arrow keys move between widgets)

* [ ] Session persistence: detach and reattach without losing state

=== 7.2 Split Pane Management

* [ ] Pane zoom (temporarily maximize one pane)

=== 7.3 Multi-Application Support

* [ ] Run multiple RAZE applications in a single terminal
* [ ] Inter-application communication via message passing
* [ ] Shared clipboard between panes
* [ ] Status line with session/pane indicators
